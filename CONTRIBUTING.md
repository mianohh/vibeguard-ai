# Contributing to VibeGuard AI

We welcome contributions to VibeGuard AI! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/vibeguard-ai.git`
3. Install dependencies: `npm install`
4. Start development server: `npm run dev`

## Development Guidelines

### Code Style
- Use TypeScript for all new code
- Follow existing code formatting and structure
- Use meaningful variable and function names
- Add comments for complex logic

### Testing
- Test your changes locally before submitting
- Ensure the application builds successfully: `npm run build`
- Verify API endpoints work as expected

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 50 characters

## Submitting Changes

1. Create a new branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Commit your changes: `git commit -m "Add your feature"`
4. Push to your fork: `git push origin feature/your-feature-name`
5. Create a Pull Request

## Areas for Contribution

- **Security Analysis**: Improve transaction risk detection
- **UI/UX**: Enhance the user interface and experience
- **Documentation**: Improve README, API docs, or code comments
- **Testing**: Add unit tests or integration tests
- **Performance**: Optimize analysis speed or memory usage

## Questions?

Feel free to open an issue for questions or discussions about potential contributions.

Thank you for contributing to VibeGuard AI! 🛡️
